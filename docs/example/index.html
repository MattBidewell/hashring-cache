Why the dots look random

They are not random at runtime. Each virtual node sits at `hash(nodeId:vnodeIndex)`, so the placement looks scattered but stays stable for the same input.

Adding weight creates more virtual nodes for that physical node. Removing a node deletes only its points, so each scattered interval it owned moves to the next surviving point clockwise.

In the ring animation, the thin dashed guide shows the previous owner only for the intervals that changed; the main ring stays focused on the new state.

The node color is consistent everywhere: dots, ring slices, and strip segments all use the same color for the same node.

This demo uses 24 base virtual nodes per weight unit so the ring stays readable. Production systems often use more, which usually smooths out the ownership split.

Remap counts are sampled from 512 demo keys, so they show the trend rather than an exact percentage of all possible keys.
