Left is hash `0`, right is `2^32 - 1`. The bar shows the current ownership map.

During a change, the thin strip above the bar shows the old owner for each remapped interval, and the highlighted section in the main bar shows the new owner.

Why the dots look random

They are not random at runtime. Each virtual node sits at `hash(nodeId:vnodeIndex)`, so the placement looks scattered but stays stable for the same input.

Adding weight creates more virtual nodes for that physical node. Removing a node deletes only its points, so only its neighboring ranges move.

In the ring animation, the thin dashed guide shows the previous owner only for the ranges that changed; the main ring stays focused on the new state.
